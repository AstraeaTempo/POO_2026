Guía Completa: Control de Flujo con if en JavaScript
En la vida real, tomamos decisiones constantemente: “Si llueve, llevo paraguas; 
si no, no”. En programación, esto se llama control de flujo, y la herramienta principal para 
lograrlo es la estructura if.
1. La base: ¿Qué es una condición?
Una condición es una expresión que solo puede tener dos resultados: 
verdadero (true) o falso (false).
Estructura del if simple
Se utiliza cuando solo nos interesa actuar si algo es cierto.
let saldo = 100;


// La condición va entre paréntesis
if (saldo > 0) {
  // El código dentro de las llaves solo se ejecuta si la condición es true
  console.log("Tienes dinero disponible.");
}

2. El camino alternativo: if...else
¿Qué pasa si la condición no se cumple? El bloque else (que significa "si no") 
define qué debe hacer el programa en ese caso. Es una bifurcación: 
o se va por un camino, o por el otro.
Ejemplo: Validación de acceso
let edad = 16;


if (edad >= 18) {
  console.log("Acceso concedido. Eres mayor de edad.");
} else {
  console.log("Acceso denegado. Eres menor de edad.");
}

3. Múltiples opciones: else if
A veces la vida no es solo "blanco o negro". 
Cuando tienes tres o más posibles resultados, encadenamos condiciones. 
El programa las evalúa de arriba hacia abajo y se detiene en la primera que sea verdadera.
Ejemplo: Sistema de semáforo

let luz = "amarillo";


if (luz === "verde") {
  console.log("Puedes avanzar.");
} else if (luz === "amarillo") {
  console.log("Precaución, detente si es posible.");
} else if (luz === "rojo") {
  console.log("¡Alto total!");
} else {
  console.log("Error: Color de luz no válido.");
}

4. Los "Jueces" de la lógica: Operadores de Comparación
Para que el if funcione, necesitamos comparar valores. En JavaScript moderno, 
usamos preferentemente la comparación estricta para evitar errores.

5. Decisiones Complejas: Operadores Lógicos
¿Qué pasa si necesitas que se cumplan dos condiciones al mismo tiempo? 
Para eso usamos los conectores lógicos.
5.1 AND (&&) - "Y"
Ambas condiciones deben ser verdaderas para que el bloque se ejecute.
JavaScript
let tieneEntrada = true;
let tieneEdadMinima = true;


if (tieneEntrada && tieneEdadMinima) {
  console.log("Puedes entrar al festival.");
}
5.2 OR (||) - "O"
Basta con que una de las condiciones sea verdadera.
let esFinDeSemana = true;
let esFeriado = false;


if (esFinDeSemana || esFeriado) {
  console.log("¡Hoy puedes dormir hasta tarde!");
}
5.3 NOT (!) - Negación
Invierte el valor de una condición. Convierte true en false y viceversa.
let estaLloviendo = false;


if (!estaLloviendo) {
  console.log("Saldré a correr.");
}

6. Buenas Prácticas (Clean Code)
Para escribir código profesional y fácil de leer, sigue estos consejos:
Usa siempre triple igual (===): El doble igual (==) hace cosas extrañas 
(como decir que el número 0 es igual al texto ""). El triple igual es más seguro.
Identación (Sangría): Siempre deja un espacio (usualmente 2 o 4 espacios) 
dentro de las llaves {}. Esto permite ver visualmente qué código pertenece al if.
No redundes con booleanos:
❌ Mal: if (esActivo === true)
✅ Bien: if (esActivo)
Llaves siempre: Aunque el código sea de una sola línea, usa las llaves {}. 
Evita errores cuando quieras agregar más líneas después.

Herramientas de consulta rápida
Antes de empezar, recuerda cómo funcionan estos métodos:

Método // ¿Qué hace? // ¿Dónde actúa?
.push() // Agrega un elemento. // Al final de la lista.
.pop() // Elimina un elemento. // Al final de la lista.
.unshift() // Agrega un elemento. // Al inicio de la lista.
.shift() // Elimina un elemento. // Al inicio de la lista.
.includes() // Revisa si algo existe. // Devuelve true o false.
.length // Cuenta los elementos. // Devuelve un número.
